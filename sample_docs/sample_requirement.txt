This document describes requirements for ShopQuick, a small e-commerce
platform targeted at small retailers who want to sell products online
quickly and with minimal setup. The system should allow users to create an
account, browse products, add items to a cart, and checkout using
credit/debit cards or common third-party payment providers. Administrators
must be able to add, edit, and remove products, and set inventory levels.

Authentication: Users should be able to register with email and
password, and optionally confirm their email address. Password reset via
email should be supported. The platform should prevent fraudulent
transactions and flag suspicious orders for review.

Catalog: The product catalog must support categories and tags, and allow
search by keyword. Search must be reasonably fast for catalogs up to
10,000 items. Product pages should display images and vary by variant
(size, color). Images should load quickly and be optimized for mobile.

Checkout: The checkout flow must calculate taxes and shipping estimates,
allow guest checkout, and provide an order confirmation email. The system
should support coupons and promotional codes applied at checkout.

Performance & Availability: The system should be responsive and
user-friendly under normal load. It should be scalable to handle sudden
traffic spikes during promotions. Backup and restore procedures should be
documented. Non-functional expectations such as "fast", "scalable" and
"user-friendly" are used in places and should be clarified.

Admin: Admins should be able to export order reports and view basic
analytics. The system should provide role-based access control for
administrative features.

Integrations: Integration with a shipping provider is required, but the
exact provider can be decided later. The API should support webhook
notifications for order events.
